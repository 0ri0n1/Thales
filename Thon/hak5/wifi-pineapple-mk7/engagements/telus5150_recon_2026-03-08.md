## Analysis (DRIFTER → VANGUARD)

1. **Closest neighbor network.** At -54 dBm, TELUS5150 is the nearest TELUS subscriber AP. This proximity means traffic from this network will be the most reliably captured in future observations.

2. **Network printer exposed.** The Epson printer (38:1A:52) is a notable finding. Consumer network printers are historically vulnerable — common issues include open printing ports (9100/JetDirect), exposed web management interfaces, SNMP information disclosure, and firmware update mechanisms that can be exploited for persistent access.

3. **Low client count unexpected.** Despite being the closest AP with strong signal and 11 IVs of data traffic, only 1 client was observed. The subscriber likely has most devices on a 5GHz band (which our 2.4GHz monitor cannot observe) or uses wired connections.

4. **Standard TELUS Actiontec deployment.** Same hardware family (FC:2B:B2) as TELUS5105. Likely identical firmware and configuration.

## Reproduction Steps

```bash
# 1. Set Pineapple monitor to Channel 11
sshpass -p <PASS> ssh root@172.16.42.1 "iw dev wlan1mon set channel 11"

# 2. Run 60s airodump-ng capture on Channel 11
sshpass -p <PASS> ssh root@172.16.42.1 \
  "rm -f /tmp/ch11-01.* 2>/dev/null; \
   airodump-ng wlan1mon --channel 11 --write /tmp/ch11 --output-format csv >/dev/null 2>&1 & \
   sleep 60; killall airodump-ng 2>/dev/null; sleep 2; cat /tmp/ch11-01.csv"

# 3. Filter results for target BSSID
# AP section: grep for FC:2B:B2:EC:D4:52
# Client section: check BSSID column for match

# 4. Targeted tcpdump for packet-level detail
sshpass -p <PASS> ssh root@172.16.42.1 \
  "iw dev wlan1mon set channel 11 && tcpdump -i wlan1mon -e -c 500 2>&1" \
  | grep -i "fc:2b:b2:ec:d4:52" | grep -v Beacon

# 5. OUI lookup
grep -i "PREFIX" /usr/share/nmap/nmap-mac-prefixes
```
