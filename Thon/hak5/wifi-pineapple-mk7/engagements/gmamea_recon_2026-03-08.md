## Analysis (DRIFTER → VANGUARD)

1. **IoT-heavy network.** All 3 associated clients are IoT/smart home devices (AltoBeam chipset, LG Innotek module, Roku streamer). No general-purpose computers or phones detected — typical of a residential network where personal devices use 5GHz band while IoT remains on 2.4GHz.

2. **TP-Link with legacy WPA fallback.** The mixed WPA2+WPA configuration is a security concern. WPA (TKIP) has known weaknesses. An attacker could force a TKIP downgrade on clients that support both modes.

3. **Cross-network device correlation.** The Bose device (28:11:A5) probes both gmamea and PHONK9876, suggesting a single household or neighbor with multiple network connections. This creates a device-tracking opportunity through probe correlation.

4. **Amazon device nearby.** An Amazon IoT device (1C:FE:2B) is probing for gmamea but not currently associated. May have been previously connected and is now searching from a different location, or is within range but connected to another network.

5. **AltoBeam chipset is a concern.** AltoBeam (Chinese chipset) devices often have minimal firmware update cycles and may run outdated, vulnerable software.

## Reproduction Steps

# 2. Run 60s airodump-ng capture on Channel 6

# 3. Filter results for target BSSID
# AP section: grep for 18:D6:C7:6C:90:3F
# Client section: check BSSID column for match

# 4. Targeted tcpdump for packet-level detail

  "iw dev wlan1mon set channel 6 && tcpdump -i wlan1mon -e -c 300 2>&1" \
  | grep -i "18:d6:c7:6c:90:3f"
